feat(ai-service): implement asynchronous AI service layer with Ollama

Implemented a robust AI service layer integrated with local Ollama using the 
gemma4:E4B model. The system provides factual summarization, keyword extraction, 
and context-aware synthesis while strictly adhering to an objective, "eyes and 
ears only" processing constraint.

## Behaviour Changes

**Before:** The system lacked an implementation for AI-driven news processing. 
AIService was defined only in documentation, with no code to handle Ollama 
connections, summarization logic, or prompt engineering.

**After:** A complete, strictly-typed AI service layer is available. It features:
- Asynchronous communication with Ollama via the official library.
- Factual summarization restricted to 1-2 concise sentences.
- Automated keyword extraction for search query generation.
- Fact-based response synthesis that forbids subjective analysis.
- Configuration via environment variables (.env).
- Diagnostic logging for all LLM interactions.

## Implementation Details

The implementation was completed across four phases, establishing a solid 
foundation and progressively adding processing logic and observability.

Phase 1: Defined AIServiceProtocol and core DTOs (NewsDTO, UserConfigDTO).
Phase 2: Integrated Ollama official library and established resilient connection handling.
Phase 3: Developed specialized system prompts for factual processing and verification.
Phase 4: Added environment variable support and diagnostic logging.
